## 6. Exploitability vs Impact Scores

**Average Scores:**
- **Exploitability:** 1.62 (median: 1.8)
- **Impact:** 4.31 (median: 3.6)
- **Impact/Exploitability Ratio:** 2.66:1

**Analysis:** On average, the impact potential is 2.66x higher than exploitability. This suggests these vulnerabilities, while requiring local access (reducing exploitability), can cause significant damage once exploited (high availability/integrity/confidentiality impact).

**Score Breakdown:**
- Exploitability 1.0 (HIGH complexity): 2 CVEs (5%)
- Exploitability 1.8 (LOW complexity, LOCAL): 37 CVEs (92.5%)
- Exploitability 3.9 (NETWORK): 1 CVE (2.5%)

## 7. Reference Tag Analysis

**Reference Statistics (across 40 CVEs):**

| Tag Type | CVE Count | Percentage | Total References |
|----------|-----------|------------|------------------|
| **Patch** | 40 | **100%** | 351 |
| **Mailing List** | 2 | 5% | 2 |
| **Vendor Advisory** | 0 | 0% | 0 |
| **Third Party Advisory** | 0 | 0% | 0 |

**Key Findings:**
- **100% of CVEs have Patch references** - excellent patch availability
- Average of **8.8 patch references per CVE** (multiple kernel version patches)
- Only 2 CVEs reference mailing lists (CVE-2022-48627, CVE-2021-47070 - Debian LTS)
- All patches link to kernel.org stable repository
- References are duplicated for both source identifiers (416baaa9... and af854a3a...)

**Reference Pattern:**
Each CVE typically has 4-8 git.kernel.org patch commits, representing fixes for different stable kernel branches (e.g., 5.4.x, 5.10.x, 5.15.x, 6.1.x, 6.5.x, 6.6-rc).

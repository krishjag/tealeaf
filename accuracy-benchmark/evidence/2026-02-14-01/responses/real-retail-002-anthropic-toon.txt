# Comprehensive Analysis of USDA Breakfast Cereal Nutritional Data

## 1. Nutrient Derivation Methods Analysis

**Distribution by Derivation Code:**

| Derivation Code | Count | Percentage | Method Description |
|----------------|-------|------------|-------------------|
| **LCSA** | 344 | 49.4% | Calculated from approximate value per serving |
| **LCCD** | 226 | 32.4% | Calculated from daily value percentage |
| **LCCS** | 72 | 10.3% | Calculated from value per serving (exact) |
| **LCGA** | 53 | 7.6% | Given by provider per 100g (approximate) |
| **LCSL** | 2 | 0.3% | Calculated from less than value |

**Key Insights:**
- **Nearly 50% use LCSA**: Manufacturers predominantly provide serving-based approximations, then USDA converts to per-100g
- **32% use LCCD**: Many micronutrients (vitamins/minerals) are back-calculated from %DV, suggesting manufacturers only report daily value percentages
- **Only 7.6% are direct measurements** (LCGA): Very few nutrients are measured directly per 100g
- **Implication**: Most nutritional data involves 2-3 calculation steps (serving ‚Üí per 100g, or %DV ‚Üí actual amount ‚Üí per 100g), introducing potential rounding errors

## 4. Duplicate Nutrient Entries Analysis

**Products with Duplicates (Same NutrientId Multiple Times):**

### **Boo Berry Halloween Cereal (FDC 2738640)** - 75 total entries, many duplicates
- Same nutrients appear **3 times each** with different contexts
- **Reason**: Three different serving contexts:
  1. Per 100g (LCSA/LCGA derivations)
  2. Per serving dry (LCSA derivations)  
  3. With milk preparation (LCSA derivations)

**Example - Protein (nutrientId 1003):**
- Value 1: 4.88g (cereal alone per 100g)
- Value 2: 3.56g (with milk per serving)
- Value 3: 5.00g (given per 100g)

### **Cheerios Breakfast Cereal (FDC 2739175)** - 86 entries
Similar pattern: dry cereal, with milk, per 100g

**Why Different Values for Same Nutrient:**
- Different **preparation states** (dry vs. with milk)
- Different **calculation methods** (LCSA approximation vs. LCGA given value)
- **Rounding** at different stages
- The database preserves all variants rather than consolidating
